Version Control:
    =>It records the changes made in the version, so that it is easy to restore the pervious.
    =>Types of Version Control:
        #Local version Control -everything is stored only in the computer locally.
        #Centralized version Control - It has a central server which stores all the files and version history.
        #Distributes Version Control - every user has the hole copy of the repository.

Problems that version control solve:
    =>Can restore the deleted version.
    =>multiple users can work on same file at same time.
    =>support branching and collaborate with other's work.
    =>go back to pervious version.

Git:
    =>Git software where you can commit,push, made in local repository.
    =>Git is the distributed version control system.
    =>where the developer can track the changes that made.
    =>can merge,commit,manage,view perious version.

GitHub:
    =>the place where commit,push,pull,merge in cloud storage
    =>can collaborate with other developer.
    =>Git does in local repository where as Github does in cloud storage.

Remote repository:
    =>It is a git repository that is stored in the another computer like (github).

Branch:
    =>It is independent line of development.
    =>that can merge with the main branch.
    =>after reviewed from the main branch it can be merged

why not directly in main branch:
    =>In the independent branch develper can debug,test and then it should be mergerd with the main branch only after the main branch accept it.
    =>so that it will be stable and complete code.

Pull Request :
    =>It is the one that is used to merge the branches.
    =>give the request to the main branch to accept the code in that branch.
